Selenium is a popular open-source framework used to automate web browsers for testing web applications
across different platforms and browsers.

üîÅ Selenium WebDriver Workflow
Here‚Äôs how a typical Selenium WebDriver test executes:
1. Test Script Execution
    - You write a test using Selenium APIs in your preferred language.
    - Example: driver.findElement(By.id("login")).click();
2. Command Translation
    - The Selenium binding converts your command into a JSON payload.
3. Protocol Communication
    - This payload is sent via HTTP to the browser driver (e.g., chromedriver.exe).
4. Driver Interpretation
    - The browser driver receives the command and translates it into native browser instructions using
      the browser‚Äôs automation engine (e.g., Chrome DevTools Protocol).
5. Browser Action
    - The browser performs the requested action (e.g., clicking a button, navigating to a URL).
6. Response Handling
    - The browser driver sends a response back to the client (e.g., success, failure, element not found).
7. Result Processing
    - Your test script processes the result and continues with the next command.

Page Object Model (POM):
The Page Object Model (POM) is a design pattern in Selenium test automation that promotes modularity,
maintainability, and reusability by separating the page structure (UI elements) from the test logic.
