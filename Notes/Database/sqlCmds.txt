Where clause:
    SELECT * FROM student WHERE marks>90;
    SELECT * FROM student WHERE marks>=90;
    SELECT * FROM student WHERE marks<=90;
    SELECT * FROM student WHERE marks IS NULL; //show NULL records only
    SELECT * FROM student WHERE marks IS NOT NULL;
    SELECT DISTINCT name FROM student;   //shows only unique records and avoid duplication
    SELECT DISTINCT * FROM student; // remove only duplicate rows

    Note: - Any comparison (=, <, >, BETWEEN, etc.) with NULL returns UNKNOWN, not TRUE or FALSE. so it will be excluded
            from result.

Logical Operators(AND/OR/NOT):
    SELECT * from student WHERE marks>90 AND name='ak'; //diplay all records which has marks>90 and name='ak'
    SELECT * from student WHERE marks>90 OR name='ak'; //diplay all records which has marks>90 or name='ak'
    SELECT * from student WHERE NOT name='aka'; //exclude all records which have name='aka'

Between/In operators: - BETWEEN 90 AND 100 means marks â‰¥ 90 and marks â‰¤ 100
    SELECT * from student WHERE marks BETWEEN 90 AND 100; // include 90 and 100 as well
    SELECT * from student WHERE marks NOT BETWEEN 90 AND 100;
    SELECT * from student WHERE marks IN(90,100); //checks if marks equal to 90, 100 then display those record only
    SELECT * from student WHERE marks NOT IN(90,100); //checks if marks not equal to 90, 100 then display those record only

Pattern Matching operators(Wild card operators):
    '%' = many characters
    '_' = single character

    SELECT * from student WHERE name LIKE 'a%'; //display only those records which name starts with 'a'
    SELECT * from student WHERE name LIKE '%a'; //display only those records which name ends with 'a'
    SELECT * from student WHERE name LIKE 'i%a'; //display only those records which name starts with 'i' and ends with 'a'
    SELECT * from student WHERE name LIKE '%a%'; //contains 'a' anywhere in the string (beginning, middle, or end).
    SELECT * from student WHERE name NOT LIKE 'a%'; //display only those records which name should NOT starts with 'a'
    SELECT * from student WHERE name LIKE '%a_'; //display only those records which name should have 2nd last char 'a' and only 1 char after 'a'
    SELECT * from student WHERE name LIKE '___'; // should have only 3 char in name

ðŸ‘‰ Group By, HAVING clause:
- Used with aggregate functions (COUNT, SUM, AVG, MIN, MAX, etc.).
- Groups rows that have the same values in specified columns into summary rows.
- HAVING clause: to filter the output from the group by clause.
- A nonâ€‘aggregated column is any column in your SELECT statement that is not inside an aggregate function like SUM(), COUNT(), AVG(), MIN(), or MAX().
- Since itâ€™s not summarized, SQL needs to know how to group it â€” thatâ€™s why it must appear in the GROUP BY clause.
Ex:
    SELECT department, AVG(salary) FROM employees GROUP BY department HAVING COUNT(*) > 10;
    - department â†’ nonâ€‘aggregated column (must be in GROUP BY).
    - AVG(salary) â†’ aggregated column (doesnâ€™t need to be in GROUP BY)

Notes:
- WHERE filters before grouping, HAVING filters after grouping.
- Always include non-aggregated columns in the GROUP BY clause.
- Useful for reports, summaries, analytics.

ðŸ‘‰ Order By clause:
- Sorts the result set by one or more columns.
- Default is ascending (ASC) order.
- You can explicitly specify ASC (ascending) or DESC (descending).
- Can sort by column names, aliases, or even expressions
Ex:
    SELECT name, salary FROM employees ORDER BY salary ASC;

ðŸ‘‰ SQL Set Operators
1. UNION
- Combines results from two queries.
- Removes duplicates by default.
Ex:
    SELECT name FROM employees UNION SELECT name FROM managers;
Result: Unique list of names from both tables.

2. UNION ALL
- Same as UNION, but keeps duplicates.
Ex:
    SELECT name FROM employees UNION ALL SELECT name FROM managers;
Result: All names, including duplicates.

3. INTERSECT ( Not supported in MySQL directly)
- Returns rows common to both queries.
- MySQL workaround: use INNER JOIN or IN.
Ex:
    SELECT name FROM employees WHERE name IN (SELECT name FROM managers);

4. EXCEPT / MINUS (Not supported in MySQL directly)
- Returns rows from the first query that are not in the second.
- MySQL workaround: use LEFT JOIN with NULL check or NOT IN.
Ex:
    SELECT name FROM employees WHERE name NOT IN (SELECT name FROM managers);

Notes:
- All queries combined with set operators must have the same number of columns and compatible data types.

ðŸ‘‰ SQL Joins:
- Joins are used to combine rows from two or more tables based on a related column.
- tables are mutually related using primary or foreign key.

1. INNER JOIN/Equi join/ simple join:
- Returns rows where there is a match in both tables.
Ex:
    SELECT e.name, d.department_name FROM employees e INNER JOIN departments d ON e.department_id = d.id;

2. LEFT JOIN (LEFT OUTER JOIN):
- Returns all rows from the left table, plus matched rows from the right table.
- If no match, NULLs are returned for right table columns
Ex:
    SELECT e.name, d.department_name FROM employees e LEFT JOIN departments d ON e.department_id = d.id;

3. RIGHT JOIN (RIGHT OUTER JOIN):
- Returns all rows from the right table, plus matched rows from the left table.
Ex:
    SELECT e.name, d.department_name FROM employees e RIGHT JOIN departments d ON e.department_id = d.id;

4. CROSS JOIN::
- Returns the Cartesian product (all combinations of rows).
Ex:
    SELECT e.name, d.department_name FROM employees e CROSS JOIN departments d;

5. FULL OUTER JOIN (Not directly supported in MySQL):
- All rows from both tables
- Emulated using UNION of LEFT JOIN and RIGHT JOIN.
Ex:
    SELECT e.name, d.department_name FROM employees e LEFT JOIN departments d ON e.department_id = d.id
    UNION
    SELECT e.name, d.department_name FROM employees e RIGHT JOIN departments d ON e.department_id = d.id;

6. SELF JOIN:
- A self join is when a table is joined with itself.
- Useful for comparing rows within the same table.
- You must use aliases to differentiate the two instances of the same table
Ex:
    SELECT e.name AS Employee, m.name AS Manager FROM employees e LEFT JOIN employees m ON e.manager_id = m.emp_id;
