üëâ What is REST Assured:
- REST Assured is a popular Java library for API automation testing. You can build a framework around it using
  Maven, TestNG/JUnit, and reusable utility classes.
- REST Assured is an open-source Java DSL (Domain Specific Language) for testing REST APIs.
- It integrates seamlessly with JUnit/TestNG and Maven.
- Supports BDD-style syntax (given(), when(), then()) for readability.
- Handles JSON/XML parsing, serialization, and deserialization easily.
- Officially supported: Java (and Groovy by extension).
- Community port: RestAssured.Net for C#.
- Alternatives for other languages and tools:
    - Python ‚Üí requests + pytest
    - JavaScript ‚Üí supertest or axios + jest/mocha
    - cypress -> Cypress isn‚Äôt just for UI automation ‚Äî it has built-in support for API testing via
                 the cy.request() command. You can use it to send HTTP requests (GET, POST, PUT, DELETE, etc.)
                 and validate responses directly inside your Cypress tests.

üëâ Do we need Rest Assured DSL chain (.given(), .when(), .then()) in Cucumber?
- Cucumber already provides the BDD keywords (Given, When, Then) in your feature files.
- Rest Assured also uses .given(), .when(), .then() ‚Äî but these are Java methods that form its DSL (Domain Specific Language).
 So, they serve different purposes:
- Cucumber keywords ‚Üí Define steps in plain English (Given I set the base URI).
- Rest Assured methods ‚Üí Build and execute HTTP requests inside your step definitions

Conclusion:
- Not mandatory to use Rest Assured‚Äôs .given()/.when()/.then() ‚Äî you could call RestAssured.get(endpoint) directly.
- Use the DSL when you want readability, maintainability, and flexibility ‚Äî especially in a BDD framework
  where the style aligns with Cucumber‚Äôs Given/When/Then.

üëâ import io.restassured.RestAssured;
- This imports the main RestAssured class.
- RestAssured ‚Üí used to send the request.
- It provides static methods to configure and send HTTP requests.
- Commonly used methods:
    - RestAssured.baseURI ‚Üí set the base URL for all requests.
    - RestAssured.given() ‚Üí start building a request (headers, params, body).
    - RestAssured.get(), post(), put(), delete() ‚Üí send requests directly.
- Example:
    RestAssured.baseURI = "https://jsonplaceholder.typicode.com";
    Response response = RestAssured.get("/posts/1");

üëâ import io.restassured.response.Response;
- This imports the Response interface/class from Rest Assured.
- Response ‚Üí used to capture and validate the response.
- Represents the HTTP response returned after making a request.
- Provides methods to:
    - response.getStatusCode() ‚Üí get HTTP status code.
    - response.getBody().asString() ‚Üí get raw response body.
    - response.jsonPath().get("key") ‚Üí extract JSON values.
    - response.getHeaders() ‚Üí access headers.
- Example:
    Response response = RestAssured.get("/posts/1");
    System.out.println("Status Code: " + response.getStatusCode());
    System.out.println("Title: " + response.jsonPath().getString("title"));

üëâ given() --> used for content type, set cookies, add auth, add param, set header info etc.
   when() --> used for get, post, put, delete
   then() --> used for validate response code, extract response, extract header cookies and response body etc.

üëâ Where .log().all() Can Be Used:
  - .log().all() is part of the validation chain (then() section), not the request execution (when()).
  - That means you cannot directly do:
      given()
        .when()
        .get("/posts/1")
        .log().all();   // ‚ùå Not valid here
    because .log() is not available immediately after .when()
- You need to place .log().all() after .then() only.
    given()
      .when()
      .get("/posts/1")
      .then()
      .log().all();   // valid
- .extract().response() ‚Üí required if you want to store the response in a Response object for further use.

üëâ What‚Äôs Inside a Response Object:
The Response object gives you everything about the HTTP response:
- Status (code, line)
- Headers
- Cookies
- Body (raw, pretty, parsed JSON/XML)
- Performance metric

1. Status Information
    - response.getStatusCode() ‚Üí HTTP status code (e.g., 200, 404, 500).
    - response.getStatusLine() ‚Üí Full status line (e.g., HTTP/1.1 200 OK).
2. Headers
    - response.getHeaders() ‚Üí All headers.
    - response.getHeader("Content-Type") ‚Üí Specific header value.
3. Cookies
    - response.getCookies() ‚Üí All cookies.
    - response.getCookie("JSESSIONID") ‚Üí Specific cookie.
4. Body
    - response.getBody().asString() ‚Üí Raw body as plain text.
    - response.getBody().prettyPrint() ‚Üí Formatted JSON/XML body.
    - response.jsonPath().get("key") ‚Üí Extract values from JSON.
    - response.xmlPath().get("tag") ‚Üí Extract values from XML.
5. Time & Performance
    - response.getTime() ‚Üí Response time in milliseconds.
    - response.getTimeIn(TimeUnit.SECONDS) ‚Üí Response time in custom unit.
6. Other Utilities
    - response.as(Class<T>) ‚Üí Deserialize body into a POJO.
    - response.path("jsonKey") ‚Üí Shortcut for extracting JSON values.
    - response.print() ‚Üí Print raw body directly.

üëâ Rest Assured Request Methods Cheat Sheet:
Request building methods in Rest Assured.

| Category        | Method / Property                        | Example Usage                                        | Purpose |
|-----------------|------------------------------------------|------------------------------------------------------|---------|
| Base Config     | `RestAssured.baseURI`                    | `RestAssured.baseURI = "https://api.example.com";`   | Set base URI for all requests |
|                 | `RestAssured.port`                       | `RestAssured.port = 8080;`                           | Set port |
|                 | `RestAssured.basePath`                   | `RestAssured.basePath = "/v1";`                      | Set base path |
|   Request Start | `given()`                                | `given().when().get("/users");`                      | Start building a request |
|   Logging       | `log().all()`                            | `given().log().all()`                                | Log full request details |
|                 | `log().headers()`                        | `given().log().headers()`                            | Log request headers |
|                 | `log().body()`                           | `given().log().body()`                               | Log request body |
|   Headers       | `header(name, value)`                    | `given().header("Content-Type", "application/json")` | Add single header |
|                 | `headers(Map<String, String>)`           | `given().headers(map)`                               | Add multiple headers |
|   Params        | `queryParam(name, value)`                | `given().queryParam("id", 1)`                        | Add query parameter |
|                 | `pathParam(name, value)`                 | `given().pathParam("userId", 10)`                    | Add path parameter |
|                 | `formParam(name, value)`                 | `given().formParam("username", "Akash")`             | Add form parameter |
| **Body**        | `body(Object)`                           | `given().body("{\"name\":\"Akash\"}")`               | Add request body (JSON/XML/String/POJO) |
| **Auth**        | `auth().basic(user, pass)`               | `given().auth().basic("admin", "password")`          | Basic authentication |
|                 | `auth().oauth2(token)`                   | `given().auth().oauth2("token123")`                  | OAuth2 authentication |
| **Methods**     | `get(endpoint)`                          | `given().when().get("/users")`                       | Send GET request |
|                 | `post(endpoint)`                         | `given().when().post("/users")`                      | Send POST request |
|                 | `put(endpoint)`                          | `given().when().put("/users/1")`                     | Send PUT request |
|                 | `delete(endpoint)`                       | `given().when().delete("/users/1")`                  | Send DELETE request |
| **Validation**  | `then().statusCode(int)`                 | `.then().statusCode(200)`                            | Validate status code |
|                 | `then().body(key, matcher)`              | `.then().body("id", equalTo(1))`                     | Validate response body |
| **Extract**     | `then().extract().response()`            | `.then().extract().response()`                       | Extract full response |
|                 | `then().extract().path("jsonKey")`       | `.then().extract().path("id")`                       | Extract specific field |


üëâ Common Ways to Create Request Body in Java

1. Using Map / HashMap
    ex: Map<String, Object> requestBody = new HashMap<>();
        requestBody.put("username", "testUser");
        requestBody.put("password", "Pass@123");

        given()
            .contentType("application/json")
            .body(requestBody)
            .post("/login");

2. External File (JSON / Excel)
ex: String requestBody = new String(Files.readAllBytes(Paths.get("src/test/resources/user.json")));

    given()
        .contentType("application/json")
        .body(requestBody)
        .post("/login");

3. POJO (Plain Old Java Object)
ex: class User {
        private String username;
        private String password;
        // getters & setters
    }

    User user = new User();
    user.setUsername("testUser");
    user.setPassword("Pass@123");

    given()
        .contentType("application/json")
        .body(user)
        .post("/login");
