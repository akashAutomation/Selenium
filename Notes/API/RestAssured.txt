- REST Assured is a popular Java library for API automation testing. You can build a framework around it using
  Maven, TestNG/JUnit, and reusable utility classes.

- Supports BDD-style syntax (given(), when(), then()) for readability.
- Handles JSON/XML parsing, serialization, and deserialization easily.
- Officially supported: Java (and Groovy by extension).
- Community port: RestAssured.Net for C#.
- Alternatives for other languages and tools:

ğŸ‘‰ Do we need Rest Assured DSL chain (.given(), .when(), .then()) in Cucumber?
- Cucumber already provides the BDD keywords (Given, When, Then) in your feature files.
- Rest Assured also uses .given(), .when(), .then() â€” but these are Java methods that form its DSL (Domain Specific Language).
 So, they serve different purposes:
- Cucumber keywords â†’ Define steps in plain English (Given I set the base URI).
- Rest Assured methods â†’ Build and execute HTTP requests inside your step definitions

Conclusion:
- Not mandatory to use Rest Assuredâ€™s .given()/.when()/.then() â€” you could call RestAssured.get(endpoint) directly.
- Use the DSL when you want readability, maintainability, and flexibility â€” especially in a BDD framework
  where the style aligns with Cucumberâ€™s Given/When/Then.

ğŸ‘‰ import io.restassured.response.Response;
- This imports the Response interface/class from Rest Assured.
- Response â†’ used to capture and validate the response.
- Represents the HTTP response returned after making a request.
- Provides methods to:
    - response.getStatusCode() â†’ get HTTP status code.
    - response.getBody().asString() â†’ get raw response body.
    - response.jsonPath().get("key") â†’ extract JSON values.
    - response.getHeaders() â†’ access headers.
- Example:

    System.out.println("Status Code: " + response.getStatusCode());
    System.out.println("Title: " + response.jsonPath().getString("title"));


ğŸ‘‰ Where .log().all() Can Be Used:
  - .log().all() is part of the validation chain (then() section), not the request execution (when()).
  - That means you cannot directly do:
      given()
        .when()
        .get("/posts/1")
        .log().all();   // âŒ Not valid here
    because .log() is not available immediately after .when()
- You need to place .log().all() after .then() only.
    given()
      .when()
      .get("/posts/1")
      .then()
      .log().all();   // valid
- .extract().response() â†’ required if you want to store the response in a Response object for further use.

ğŸ‘‰ Whatâ€™s Inside a Response Object:
The Response object gives you everything about the HTTP response:
- Status (code, line)
- Headers
- Cookies
- Body (raw, pretty, parsed JSON/XML)
- Performance metric

1. Status Information
    - response.getStatusCode() â†’ HTTP status code (e.g., 200, 404, 500).
    - response.getStatusLine() â†’ Full status line (e.g., HTTP/1.1 200 OK).
2. Headers
    - response.getHeaders() â†’ All headers.
    - response.getHeader("Content-Type") â†’ Specific header value.
3. Cookies
    - response.getCookies() â†’ All cookies.
    - response.getCookie("JSESSIONID") â†’ Specific cookie.
4. Body
    - response.getBody().asString() â†’ Raw body as plain text.
    - response.getBody().prettyPrint() â†’ Formatted JSON/XML body.
    - response.jsonPath().get("key") â†’ Extract values from JSON.
    - response.xmlPath().get("tag") â†’ Extract values from XML.
5. Time & Performance
    - response.getTime() â†’ Response time in milliseconds.
    - response.getTimeIn(TimeUnit.SECONDS) â†’ Response time in custom unit.
6. Other Utilities
    - response.as(Class<T>) â†’ Deserialize body into a POJO.
    - response.path("jsonKey") â†’ Shortcut for extracting JSON values.
    - response.print() â†’ Print raw body directly.

Request building methods in Rest Assured.

| Category        | Method / Property                        | Example Usage                                        | Purpose |
|-----------------|------------------------------------------|------------------------------------------------------|---------|

|                 | `RestAssured.port`                       | `RestAssured.port = 8080;`                           | Set port |

|   Headers       | `header(name, value)`                    | `given().header("Content-Type", "application/json")` | Add single header |

|                 | `pathParam(name, value)`                 | `given().pathParam("userId", 10)`                    | Add path parameter |
